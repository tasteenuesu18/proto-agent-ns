# Proto Agent NS (実況プレイ・エージェント)

ユーザーの質問に対して、AIが裏側でどのように考え、どのようなツールを使用して回答を生成しているのか（思考プロセスとツールの実行結果）を可視化することで、AIエージェントの仕組みを学ぶための学習用プロトタイプです。

## プロジェクトの目的と意図

このプロジェクトは、AIエージェントが自律的に問題解決を行う様子を「実況プレイ」のように観察できるように設計されています。通常、ユーザーはAIからの最終的な回答しか見ることができませんが、本アプリケーションでは以下の要素をリアルタイムでストリーミング表示します。

*   **ユーザーの入力**: ユーザーが知りたい情報や指示。
*   **AIの思考（Text）**: AIが次に何をすべきか考える過程。
*   **ツールの呼び出し（Tool Call）**: AIが外部システム（検索エンジンなど）にアクセスする準備。
*   **ツールの実行結果（Tool Result）**: 外部システムから得られた生データ。
*   **最終的な回答**: AIが情報を整理してユーザーに示す最終結果。

これにより、LLM（Large Language Model）がどのように「計画」し、「行動」し、「観察」し、「軌道修正」しているのかを直感的に理解することができます。

## 技術スタックと構成

*   **フロントエンド**: [Next.js](https://nextjs.org/) (App Router), React, Tailwind CSS
*   **AI 制御ライブラリ**: [Vercel AI SDK](https://sdk.vercel.ai/docs) (`ai`, `@ai-sdk/react`, `@ai-sdk/google`)
*   **言語モデル**: Google Gemini (`gemini-3-flash-preview`)
    *   Google AI Studioから提供されるモデルを使用し、費用とパフォーマンスのバランスを取っています。
*   **外部検索ツール**: [Tavily API](https://tavily.com/) (`@tavily/core`)
    *   AI向けに最適化された検索エンジンで、迅速かつ正確な検索結果をLLMに提供します。

## アーキテクチャとツールの定義

アプリケーションは大きく分けて、フロントエンドのUIコンポーネントと、バックエンドのAPIルートで構成されています。

### 1. バックエンド API (`src/app/api/chat/route.ts`)

Vercel AI SDKの `streamText` を使用して、モデルとの対話を管理し、その結果をストリームとしてクライアントに返します。

*   **モデルの設定**: `google('gemini-3-flash-preview')` を指定。
*   **最大ループ数の制限**: 無限ループによる課金を防ぐため、`stopWhen: stepCountIs(5)` でエージェントの自律ループを最大5ステップに制限しています。
*   **システムプロンプト (System Prompt)**: エージェントの役割（親切で有能なAIアシスタント）と、回答に検索ツールを用いるべき指示を与えています。
*   **ツールの定義 (Tools)**:
    *   `search`: ウェブ上の最新情報を検索するツール。
    *   **入力スキーマ (`inputSchema`)**: `query` (検索キーワード) を受け取るように定義 (`zod` を使用)。
    *   **実行関数 (`execute`)**: 実際に Tavily API を呼び出し、検索結果を取得して LLM に返します。

### 2. フロントエンド UI (`src/app/page.tsx`)

Vercel AI SDK の `useChat` フックを利用して、APIルートとのストリーミング通信と状態管理を行います。

*   **メッセージレンダリング**: 新しい AI SDK (v5) の `UIMessage` 形式に対応し、`m.parts` を反復処理することで、テキストとツール呼び出しを適切に分離して表示します。
*   **プロセスの可視化**:
    *   `part.type === 'text'` の場合: AIの思考や最終回答として表示。
    *   `part.type === 'tool-invocation'` の場合: ツールの実行前段階（引数など）と、実行完了後の結果（出力やエラー）、さらに進行中の状態（ローディング）を分かりやすく区別して描画します。
*   **ステータス管理**: `status` が `submitted` や `streaming` の場合は入力フィールドを無効化するなど、UIの制御を行っています。

## セットアップと起動

### 前提条件

*   Node.js (LTSバージョン推奨)
*   Google AI Studio (Gemini) APIキー
*   Tavily APIキー

### 環境変数の設定

プロジェクトのルートディレクトリに `.env.local` ファイルを作成し、以下の変数を設定してください。

```env
GOOGLE_GENERATIVE_AI_API_KEY="あなたの_GEMINI_API_キー"
TAVILY_API_KEY="あなたの_TAVILY_API_キー"
```

### インストールと起動

```bash
# 依存関係のインストール
npm install

# 開発サーバーの起動
npm run dev
```

ブラウザで [http://localhost:3000](http://localhost:3000) を開き、エージェントの実況プレイを確認してください。
