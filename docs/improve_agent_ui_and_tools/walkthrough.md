# 追加改修のウォークスルー：Agent/LLM機能・UIの分離と天気API安定化

## 概要
ユーザーの追加要望に基づき、「LLM（脳）」と「Agent（システム・手足）」の役割をダッシュボード上で視覚的に明確に分ける改修を実施しました。また、安定して回答を得られるよう天気取得用の外部APIを変更しました。

## 実装した変更内容

### 1. 天気APIの変更 (`wttr.in` の利用)
- `src/app/api/chat/route.ts` 内の `get_weather` ツールの実装を変更し、これまでの Open-Meteo および Geocoding API の複雑な処理から、非常にシンプルで確実なレスポンスを返す `wttr.in` (JSONフォーマット) に置き換えました。
- これにより、ユーザーからの「天気を教えて」という要求に対して安定して現在の気温や天候（晴れ、曇りなど）を取得できるようになりました。

### 2. LLM と Agent のデータ分離とキャプチャ
- 同様に `route.ts` の `onStepFinish` メソッド内で、Vercel AI SDK から取得できる **「LLMへ送信した生プロンプト（`request.messages`）」** と **「LLMが生成した生レスポンス（`response.messages`）」** を追加でキャプチャし、フロントエンドに返却する `StepData` 型に含めるようにしました。

### 3. アクターの境界を明確にするUI改修
- `src/app/page.tsx` を修正し、以下のUI変更を行いました。
  - **入力の制限**: 自由に入力できるテキストボックスを削除し、用意された3つのサジェストボタンからのみリクエストを行えるようにしました。
  - **実行シーケンスの視覚化**: メイン画面の「実行シーケンス概要」において、各プロセスの実行者が誰であるか（青：User、紫：LLM、緑：Agent System）のアイコンとバッジを配置しました。これにより、「LLMが推論を行いツールの使用を指示する」→「Agentが実際のツールを実行する」という役割の境界がステップごとに明確になります。
  - **タブ2 (LLM送受信ログ) の2ペイン化**: 以前は単一のログだったものを、左側に「Agent → LLM (Request Prompt)」、右側に「LLM → Agent (Response Generation)」という2カラム構成に変更しました。Agentがどんな情報を脳（LLM）へ送り、脳（LLM）がどんな思考テキストやツール呼び出し命令を返したかが一目で理解できるようになります。

## テスト結果 (Verification)
- ローカル環境のブラウザ（`localhost:3000`）にて、「今日の東京の天気を教えて」のサジェストボタンを押下し、以下を確認しました。
  1.  `wttr.in` を使った天気データ（Tokyo, 18℃, 晴れ など）が取得され、最終出力に反映されること。
  2.  「2. LLM送受信ログ」タブで、Agentからのリクエストペイロードと、LLMからの生成テキスト・ツール指示が左右に分割されて正しく表示されること。
  3.  「実行シーケンス概要」で User -> LLM -> Agent System という各アクターの処理フローがアイコン付きで分かりやすく表示されること。
  
以上で追加機能の開発とテストは完了です。
